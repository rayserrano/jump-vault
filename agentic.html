Agentic

The consumer talks to the platform now.

Ask what there is to know, and the agent answers from the semantic manifest. Ask how to think about a metric, and it explains from the definitions it was served. The question itself gets answered from governed, centralized metrics, the same ones every dashboard compiles. And when the analysis is settled, the agent drives the BI tool over the same open standard, materializes the conversation as a shipped dashboard, and the loop begins again. This page describes that loop, what it changes about BI, what it makes valuable, what it lets Jump sell, and where it goes next.

The Loop

From spoken question to shipped dashboard.

The loop has four stations. Discovery: the semantic manifest is the menu, so the first thing an agent can do is tell the consumer what the platform knows and what each thing means. Education: the definitions carry their own reasoning, so the agent can teach how to think about a metric before anyone computes it. Analysis: every answer comes from the governed layer, so the conversation and the company dashboard cannot disagree. Delivery: when the consumer is satisfied, the agent drives the BI surface through the Model Context Protocol, builds the dashboard from the same definitions, and ships it. The consumer never left the conversation, and everything produced along the way is governed.

What BI Becomes

The dashboard becomes the receipt.

Self-serve analytics kept one promise poorly: it showed people a catalog of tables and wished them luck, and a year later there were four definitions of revenue. The agent replaces that failed middle. Discovery becomes a conversation, education comes from the definitions themselves, and answers come from one governed place. The dashboard survives, but demoted, from the product to the receipt: a persistent surface the agent materializes when an analysis is worth keeping, cheap to produce, disposable without ceremony, and regenerable at any time from the same definitions that produced it. The BI tool does not disappear in this future. It becomes the printer.

What Becomes Scarce

The agent amplifies whatever definitions exist.

An agent makes good semantics fast and bad semantics faster. Four drifting definitions of revenue at conversation speed is not self-serve; it is confident wrong answers at scale. So the layer underneath the agent gets more valuable in this future, not less. The governed definitions become the ground truth every answer flows from, the appreciating asset of the whole platform, and the thing worth staffing seriously. Every claim on the Architecture and Engineering pages was building toward this sentence: the intelligence layer is exactly as good as the governed layer it reads.

The UMM

The best business analysis, condensed into a library.

The Universal Data Model consolidated the entities every business shares. The Universal Metrics Model does the same for the questions. It is a cataloged library of metric definitions, and each entry carries its formula, the canonical analysis it condenses with the source named, the thresholds that make it meaningful, and its decomposition, with headline indicators breaking into the component measures that explain them. Nobody re-derives a revenue metric from scratch. The best existing analysis of it is already condensed, cited, and declared in semantic code.

The library travels for a mechanical reason: every definition is written against the universal entities rather than any one company’s tables. A renewal rate is a renewal rate whether the member holds a gym card or a season ticket, so the analysis only has to be written once, and it lands wherever the entities exist. For this platform, that changes what onboarding means. Stamping a club stamps the metric library with the mart: day-one dashboards, and an agent that arrives already educated, because the briefing came with the platform.

For the agent, the library is the difference between a query engine and an analyst. A metric that knows its own decomposition can be explained level by level, and a metric that cites its canon can defend itself.

The Intake

The library grows by demand.

Sometimes the consumer asks something the layer cannot answer, and that is the healthiest moment in the loop. An unanswerable question is not a failure; it is the intake funnel. It becomes the ticket for the next governed metric, the guide on the Engineering page is the mechanism that mints it, and the next conversation can answer what the last one could not. The semantic layer grows by demand instead of by speculation, and the agent serves at both ends of the process, as the consumer of the library and as the backlog for it.

The Tenth Product

The data platform becomes something to sell.

Every ingredient this page assembled is also a package. The governed semantic layer, the metric library stamped per club, the agent connected through MCP, and tenancy isolated by construction add up to a per-club offering: each club gets its own agent, educated on its own mart, fluent in its own metrics, and blind to everyone else’s. The nine applications earn revenue on the fan experience. This one earns it on what the nine already produce, and onboarding does not change shape to deliver it: stamping a club stamps the mart, the metrics, and now the agent, the same press with one more output.

The offering is also defensible for the same reason it is possible. An agent is exactly as good as the governed layer it reads, so nobody sells this without building the platform first. The role description said the platform is becoming a full product. Agents as a Service takes that sentence all the way: the platform itself becomes the thing on the price list.

The Horizon

Everything on this page still waits to be asked.

There are three eras of intelligence on data. The first is agents you invoke: ask, think, answer, sleep. Everything above lives there, and it is real and it is good. The second is intelligence you compile: the models, the rules, and the metric libraries this site has been assembling page by page. The third is cognition that streams: a continuous process standing on the business event stream, aware of every fan transaction as it happens, watching everything cheaply and thinking only when something deserves it. Today agents visit the data. The horizon is an intelligence that lives on it.

The industry already ships this pattern. Confluent sells it as Streaming Agents: event-driven agents built and orchestrated natively on Flink, embedded in the stream itself, in open preview since 2025 and production-grade in the current platform. The shape is standard: the event backbone carries what happens, stateful stream processing holds each club’s context as keyed state, a salience gate decides what deserves thought, the language model is invoked only at that gate, and every output compiles back into the governed layer as rules, features, and materializations. The economics are the physiology: watching is stream-processing cheap, and tokens are spent only where thinking happens.

Build note: the horizon diagram lands here in the polish pass, drawn with the loop diagram in one visual language: event backbone, stream state, salience gate, selective thought, compiled outputs.

Two properties of this platform carry over intact. Keyed state gives every club its own partition, isolated by construction, the streaming twin of the mart boundary. And the agent on the stream reads the same governed definitions as the agent in the conversation, so the third era inherits the semantic layer rather than escaping it. The pages before this one are the prerequisites of that future. The author of this study has designed and operated event-driven agent systems on this exact stack.
